# 고전 ML 모델의 입력 특성 설명

## ✅ 현재 구현: 3D Array의 6가지 특성 (활성화됨)

### 데이터 형식: **(29, 64, 6)** ✨

- **29**: 생성된 시간 윈도우 개수
- **64**: 각 윈도우의 샘플 개수 (WINDOW_SIZE=64)
- **6**: 각 샘플마다의 특성 개수 ← **✨ 새로 추가됨!**

### 6가지 특성의 상세 설명

각 윈도우의 64개 샘플에 대해 다음 6개 특성을 생성합니다:

| 인덱스 | 특성명 | 수식/설명 | 용도 | 계산 범위 |
|--------|--------|----------|------|---------|
| [w][s][0] | **vibration** | 원본 진동값 (정규화됨) | 기본 신호 | 각 샘플 s |
| [w][s][1] | **RMS** | √(mean(vibration²)) | 진동 에너지 수준 | **윈도우 w 전체** ※ |
| [w][s][2] | **Peak** | max(\|vibration\|) | 최대 진동 크기 | **윈도우 w 전체** ※ |
| [w][s][3] | **Crest Factor** | Peak / RMS | 신호 충돌성 | **윈도우 w 전체** ※ |
| [w][s][4] | **Kurtosis** | 통계적 첨도 (꼬리) | 이상 임펄스 감지 | **윈도우 w 전체** ※ |
| [w][s][5] | **Skewness** | 통계적 왜도 (비대칭) | 신호 비대칭성 | **윈도우 w 전체** ※ |

✨ **중요**: 특성 1~5는 윈도우 전체(64개 샘플)에서 계산되므로, 같은 윈도우의 모든 샘플이 동일한 값을 가집니다.

### 예시: 윈도우 0의 데이터 구조

```python
Window 0 (shape: 64, 6):
[
  [0.0064,   0.1234,  0.5678,  5.1234,  0.2345,  -0.1234],  # Sample 0
  [-0.0184,  0.1234,  0.5678,  5.1234,  0.2345,  -0.1234],  # Sample 1
  [-0.0095,  0.1234,  0.5678,  5.1234,  0.2345,  -0.1234],  # Sample 2
  ...
  [0.0050,   0.1234,  0.5678,  5.1234,  0.2345,  -0.1234],  # Sample 63
]

주목:
- Column 0: 각 샘플의 서로 다른 vibration 값
- Columns 1-5: 윈도우 전체에서 계산되어 모든 행에 동일한 값
```

## 모델 입력 형식

### 전체 파이프라인

```
원본 CSV (각 파일 500 샘플)
    ↓ [데이터 로드 & 병합]
DataFrame (5000, 3) = [time, vibration, label]
    ↓ [정규화]
Normalized DataFrame
    ↓ [윈도우 생성]
3D Array (29, 64, 1)
    ↓ [✨ 통계 특성 추가] ← 새로 추가!
3D Array (29, 64, 6) ✨
```

### 각 모델별 입력 형식 (학습 시)

| 모델 | 입력 포맷 | 내부 처리 | 최종 사용 |
|-----|-------------------|---------|----------|
| **RandomForest** | (29, 64, 6) | → (29, 384) 평탄화 | sklearn 2D 요구 |
| **IsolationForest** | (29, 64, 6) | → (29, 384) 평탄화 | sklearn 2D 요구 |
| **OneClassSVM** | (29, 64, 6) | → (29, 384) 평탄화 | sklearn 2D 요구 |
| **Autoencoder** | (29, 64, 6) | → (29, 384) 평탄화 | Linear layers |
| **LSTM** | (29, 64, 6) | **유지** | RNN (batch, seq, features) |

✨ **차이점**: Classical ML + Autoencoder는 평탄화하여 **384개 특성** 사용, LSTM은 **3D 구조 유지**



```python
# 3D Array에서 첫 번째 특성만 추출
X_vibration_only = X[:, :, 0]  # (29, 64)

# 2D로 평탄화
X_flat = X_vibration_only.reshape(29, 64)  # (29, 64)

# 또는 더 간단하게
X_flat = X[:, :, 0].reshape(29, -1)

RandomForest.fit(X_flat)  # ✓ 진동값 64개만 사용
```

### 방법 2: 통계 특성도 포함 (권장!)

```python
# 모든 6개 특성 사용 (더 정보가 많음)
X_all = X[:, :, :].reshape(29, -1)  # (29, 384)

# 또는 윈도우별 집계 사용
X_agg = aggregate_features(X)  # (29, 24)

RandomForest.fit(X_all)  # 또는 X_agg
```

## 성능 비교

| 방식 | 입력 크기 | 특성 수 | 장점 | 단점 |
|------|----------|--------|------|------|
| **진동값만** | (29, 64) | 64 | 간단함 | 정보 부족 |
| **모든 샘플** | (29, 384) | 384 | 모든 정보 포함 | 차원 너무 많음 |
| **윈도우 집계** | (29, 24) | 24 | 균형적, 효율적 | 일부 정보 손실 |

## 결론

### ❓ 고전 ML에서 진동값만 입력하면 되나요?

**기술적으로는 가능하지만, 권장하지 않습니다!**

- ✅ 가능: 진동값(특성 0)만 추출해서 사용 가능
- ⚠️ 비권장: 다른 통계 특성들(RMS, Peak, Curst, Kurtosis, Skewness)도 유용한 정보 포함
- 🎯 권장: **모든 6개 특성을 사용** (또는 윈도우별 집계)

### 이유:

1. **RMS**: 진동 에너지 수준 → 이상 신호의 강도
2. **Peak**: 최대 진동 → 충격 감지
3. **Crest Factor**: Peak/RMS → 신호의 "스파이크" 특성
4. **Kurtosis**: 이상 임펄스의 첨도 → 베어링 손상 등 감지에 탁월
5. **Skewness**: 신호 비대칭성 → 비선형 이상 감지

### 권장 사용 방식:

```python
# 1️⃣  모든 특성 사용 (간단함)
X_input = X.reshape(X.shape[0], -1)  # (29, 384)

# 2️⃣  윈도우 집계 사용 (효율적) ← 추천!
X_input = aggregate_window_features(X)  # (29, 24)

# 3️⃣  진동값만 사용 (정보 손실)
X_input = X[:, :, 0].reshape(X.shape[0], -1)  # (29, 64)

model.fit(X_input)
```
