# 고전 ML 모델의 입력 특성 설명

## 현재 3D Array의 6가지 특성

3D Array 형태: **(29, 64, 6)**
- 29: 시간 윈도우
- 64: 각 윈도우의 샘플 개수
- **6: 특성 개수** ← 이게 뭐냐가 핵심!

### 6가지 특성의 구성:

| 인덱스 | 특성명 | 설명 | 용도 |
|--------|--------|------|------|
| [w][s][0] | **vibration** | 원본 진동값 (정규화됨) | 기본 신호 |
| [w][s][1] | **RMS** | 평균 제곱근 | 진동 에너지 수준 |
| [w][s][2] | **Peak** | 최대 절댓값 | 최대 진동 크기 |
| [w][s][3] | **Crest Factor** | Peak / RMS | 신호 충돌성 |
| [w][s][4] | **Kurtosis** | 첨도 (꼬리의 두께) | 이상 임펄스 감지 |
| [w][s][5] | **Skewness** | 왜도 (비대칭성) | 신호 비대칭성 |

### 예시:

```python
3D Array[0]:  # Window 0
[
  [0.0064,   0.1234,  0.5678,  5.1234,  0.2345,  -0.1234],  # Sample 0
  [-0.0184,  0.1234,  0.5678,  5.1234,  0.2345,  -0.1234],  # Sample 1
  [-0.0095,  0.1234,  0.5678,  5.1234,  0.2345,  -0.1234],  # Sample 2
  ...
  [0.0050,   0.1234,  0.5678,  5.1234,  0.2345,  -0.1234],  # Sample 63
]

shape: (64, 6)

각 행(샘플): 6개 특성
각 열(특성): 64개 값 (시간 시리즈)
```

## 고전 ML 모델의 입력 방식

### ❌ 고전 ML은 3D 데이터를 직접 사용할 수 없음

```python
# 이건 안됨 (3D 배열)
RandomForest.fit(X)  # X.shape = (29, 64, 6)  ← ERROR!
```

### ✅ 해결 방법 1: 2D로 평탄화 (현재 사용 중)

```python
# 각 윈도우를 1D 벡터로 변환
X_2d = X.reshape(29, 64 * 6)  # (29, 384)
# 384 = 64 샘플 × 6 특성

RandomForest.fit(X_2d)  # ✓ 작동함!

# 의미: 각 윈도우의 64 샘플 × 6 특성 = 384개 피처
```

샘플 데이터:
```
Window 0: [0.0064, -0.0184, ..., -0.1234, 0.2345, 5.1234, ...]  # 384 값
Window 1: [0.0145, 0.0267, ..., 0.0128, -0.0567, 4.9876, ...]   # 384 값
...
```

### ✅ 해결 방법 2: 윈도우당 집계 특성 (권장!)

각 윈도우의 특성을 **평균, 최대, 최소 등으로 집계**:

```python
# 각 윈도우별로 6개 특성의 통계를 계산
X_aggregated = X.reshape(29, 64, 6)  # (29, 64, 6)

aggregated_features = []
for window in X:
    # 각 특성별 평균 (64개 샘플 → 1개 값으로 축약)
    mean = window.mean(axis=0)      # (6,)
    std = window.std(axis=0)        # (6,)
    max = window.max(axis=0)        # (6,)
    min = window.min(axis=0)        # (6,)
    
    # 모두 연결
    features = np.concatenate([mean, std, max, min])  # (24,)
    aggregated_features.append(features)

X_aggregated_2d = np.array(aggregated_features)  # (29, 24)
RandomForest.fit(X_aggregated_2d)  # ✓ 훨씬 더 효율적!
```

## 진동값만 사용하려면?

만약 **진동값(특성 0)만** 사용하고 싶다면:

### 방법 1: 진동값만 추출

```python
# 3D Array에서 첫 번째 특성만 추출
X_vibration_only = X[:, :, 0]  # (29, 64)

# 2D로 평탄화
X_flat = X_vibration_only.reshape(29, 64)  # (29, 64)

# 또는 더 간단하게
X_flat = X[:, :, 0].reshape(29, -1)

RandomForest.fit(X_flat)  # ✓ 진동값 64개만 사용
```

### 방법 2: 통계 특성도 포함 (권장!)

```python
# 모든 6개 특성 사용 (더 정보가 많음)
X_all = X[:, :, :].reshape(29, -1)  # (29, 384)

# 또는 윈도우별 집계 사용
X_agg = aggregate_features(X)  # (29, 24)

RandomForest.fit(X_all)  # 또는 X_agg
```

## 성능 비교

| 방식 | 입력 크기 | 특성 수 | 장점 | 단점 |
|------|----------|--------|------|------|
| **진동값만** | (29, 64) | 64 | 간단함 | 정보 부족 |
| **모든 샘플** | (29, 384) | 384 | 모든 정보 포함 | 차원 너무 많음 |
| **윈도우 집계** | (29, 24) | 24 | 균형적, 효율적 | 일부 정보 손실 |

## 결론

### ❓ 고전 ML에서 진동값만 입력하면 되나요?

**기술적으로는 가능하지만, 권장하지 않습니다!**

- ✅ 가능: 진동값(특성 0)만 추출해서 사용 가능
- ⚠️ 비권장: 다른 통계 특성들(RMS, Peak, Curst, Kurtosis, Skewness)도 유용한 정보 포함
- 🎯 권장: **모든 6개 특성을 사용** (또는 윈도우별 집계)

### 이유:

1. **RMS**: 진동 에너지 수준 → 이상 신호의 강도
2. **Peak**: 최대 진동 → 충격 감지
3. **Crest Factor**: Peak/RMS → 신호의 "스파이크" 특성
4. **Kurtosis**: 이상 임펄스의 첨도 → 베어링 손상 등 감지에 탁월
5. **Skewness**: 신호 비대칭성 → 비선형 이상 감지

### 권장 사용 방식:

```python
# 1️⃣  모든 특성 사용 (간단함)
X_input = X.reshape(X.shape[0], -1)  # (29, 384)

# 2️⃣  윈도우 집계 사용 (효율적) ← 추천!
X_input = aggregate_window_features(X)  # (29, 24)

# 3️⃣  진동값만 사용 (정보 손실)
X_input = X[:, :, 0].reshape(X.shape[0], -1)  # (29, 64)

model.fit(X_input)
```
