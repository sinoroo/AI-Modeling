"""
전처리 후 3D Array 상세 분석

이 문서는 전처리 후 생성되는 3D Array의 구조와 포함된 정보를 상세히 설명합니다.
"""

# ============================================================================
# 전처리 후 3D Array 구조 (154, 64, 1)
# ============================================================================

"""
3D Array Shape: (154, 64, 1)
│
├─ 차원 1 (154): 윈도우 개수 (Number of Windows)
├─ 차원 2 (64): 각 윈도우의 샘플 개수 (Window Size)
└─ 차원 3 (1): 특성 개수 (Number of Features)

예시 구조:
array([[[ 0.0064],    ← Window 0, Sample 0, Feature 0 (vibration)
        [-0.0184],    ← Window 0, Sample 1, Feature 0
        [-0.0095],    ← Window 0, Sample 2, Feature 0
        ...           ← Window 0, Samples 3-62
        [ 0.0050]],   ← Window 0, Sample 63 (마지막 샘플)
       
       [[ 0.0145],    ← Window 1, Sample 0, Feature 0
        [ 0.0267],    ← Window 1, Sample 1, Feature 0
        ...           ← Window 1, Samples 2-62
        [-0.0089]],   ← Window 1, Sample 63
       
       ...            ← Windows 2-152
       
       [[-0.0032],    ← Window 153 (마지막 윈도우), Sample 0
        [ 0.0078],    ← Window 153, Sample 1
        ...           ← Windows 153, Samples 2-62
        [ 0.0145]]]) ← Window 153, Sample 63 (마지막)


print(array.shape)  # (154, 64, 1)
print(array[0])     # (64, 1) - 첫 번째 윈도우
print(array[0][0])  # (1,) - 첫 번째 윈도우의 첫 샘플
print(array[0][0][0])  # 0.0064 - 실제 진동값
"""


# ============================================================================
# 각 차원의 의미
# ============================================================================

"""
┌──────────────────────────────────────────────────────────────────────────┐
│ 차원 1: Window Index (윈도우 인덱스) - 0 ~ 153
└──────────────────────────────────────────────────────────────────────────┘

의미:
- 전체 4990개의 원본 샘플을 64개씩 겹치는 윈도우로 나눔
- 겹침 정도: 50% (overlap)
- 단계 크기: 32 (step = window_size * (1 - overlap) = 64 * 0.5 = 32)

윈도우 작성 과정:
┌─────────────────────────────────────────────────────────────────────┐
│ Original time-series data: [s0, s1, s2, ..., s4989]               │
│ (4990 samples)                                                     │
│                                                                     │
│ Window 0: [s0, s1, ..., s63] (첫 64개 샘플)                        │
│           ↓ step=32 이동                                           │
│ Window 1: [s32, s33, ..., s95] (s32부터 64개)                      │
│           ↓ step=32 이동                                           │
│ Window 2: [s64, s65, ..., s127]                                    │
│           ↓ step=32 이동                                           │
│ ...                                                                │
│           ↓                                                        │
│ Window 153: [s4926, s4927, ..., s4989] (마지막 윈도우)            │
└─────────────────────────────────────────────────────────────────────┘

결과:
- 총 154개 윈도우 생성
- 각 윈도우는 64개 연속 샘플 포함
- 인접한 윈도우들은 32개 샘플씩 겹침 (50% overlap)


┌──────────────────────────────────────────────────────────────────────────┐
│ 차원 2: Sample Index (샘플 인덱스) - 0 ~ 63
└──────────────────────────────────────────────────────────────────────────┘

의미:
- 각 윈도우 내에서의 샘플 위치
- Window_size = 64로 설정됨 (config.py에서 설정 가능)

시간축 표현:
array[w][0]  → 시간 t0 (윈도우 시작)
array[w][1]  → 시간 t0 + Δt (1번째 샘플 이후)
array[w][2]  → 시간 t0 + 2Δt (2번째 샘플 이후)
...
array[w][63] → 시간 t0 + 63Δt (윈도우 끝)

여기서 Δt = 1/sampe_rate = 1/4000 = 0.00025 초

예시:
Window 0의 시간 범위:
- 첫 샘플: 0.00000 초
- 마지막 샘플: 0.01575 초 (63 * 0.00025)

Window 1의 시간 범위:
- 첫 샘플: 0.00800 초 (32 * 0.00025)
- 마지막 샘플: 0.02375 초


┌──────────────────────────────────────────────────────────────────────────┐
│ 차원 3: Feature Index (특성 인덱스) - 0 (1개만)
└──────────────────────────────────────────────────────────────────────────┘

의미:
- 현재는 1개 특성만 (vibration)
- CSV 파일에서 추출한 시간-진동 데이터
- 이전 단계에서 vibration 컬럼만 필터링됨

array[w][s][0]
        ↑ 이 0은 vibration feature를 의미
        
예시:
array[0][0][0]  # Window 0, Sample 0의 진동값
                # 값: -0.006486 (정규화된 진동 데이터)

만약 여러 특성이 있다면:
array[w][s][0]  # Feature 0 (vibration)
array[w][s][1]  # Feature 1 (temperature) - 현재는 없음
array[w][s][2]  # Feature 2 (pressure) - 현재는 없음
...
"""


# ============================================================================
# 데이터 정표화 (Normalization) 과정
# ============================================================================

"""
전처리 파이프라인:

1. 원본 데이터 로드 (CSV)
   DataFrame (4990, 3) [time, vibration, label]
   vibration: -0.0065 ~ 0.1523 (자유로운 범위)
        ↓

2. 결측치 처리
   - 선형 보간법 사용
   - 결측값이 있으면 주변 값으로 보간
   - 이 데이터는 결측치 없음
        ↓

3. 이상치 처리 (Outlier Handling)
   - IQR (Interquartile Range) 방법
   - Q1 = 25 percentile
   - Q3 = 75 percentile
   - IQR = Q3 - Q1
   - 범위: [Q1 - 1.5*IQR, Q3 + 1.5*IQR]
   - 범위 밖의 값은 제거 또는 클립
        ↓

4. 정규화 (Normalization)
   - 방법: StandardScaler (Standardization)
   - 수식: x_normalized = (x - mean) / std
   - 결과 범위: 대략 [-3, 3]
   - 장점: 평균 0, 표준편차 1로 정규분포화
   
   예시:
   Original: -0.0065, -0.0184, -0.0095, ...
   Normalized: -0.0234, -0.0667, -0.0344, ...
        ↓

5. 시계열 윈도우 생성 (Time-series Windowing)
   - Window size: 64 (연속 샘플)
   - Overlap: 50%
   - Step: 32 (64 * 0.5)
   
   (4990, 1) → (154, 64, 1)
   
   4990개 샘플을 64개씩 자르되, 32개 스텝으로 이동
         ↓

6. 최종 3D Array
   Shape: (154, 64, 1)
   Content: 정규화된 진동 데이터
   값 범위: 약 [-3, 3]
   데이터 타입: float32
   메모리: 154 * 64 * 1 * 4 bytes = 39.4 KB
"""


# ============================================================================
# 각 윈도우가 나타내는 시간 간격
# ============================================================================

"""
Config 설정:
- Sample Rate: 4000 Hz
- Window Size: 64 samples
- Overlap: 50%
- Step Size: 32 samples

시간 계산:
- 샘플 간 시간 간격: 1/4000 = 0.00025 초 = 0.25 ms

각 윈도우의 지속 시간:
- 64 샘플 × 0.00025 초/샘플 = 0.016 초 = 16 ms

윈도우 간 시간 간격:
- 32 샘플 × 0.00025 초/샘플 = 0.008 초 = 8 ms

타임라인:

전체 데이터: 4990 샘플 × 0.00025 초/샘플 = 1.2475 초

Window 0:  ├─ 0.000 ~ 0.016 초
           │
           └─ 시간 간격: 8ms
           
Window 1:       ├─ 0.008 ~ 0.024 초
                │
                └─ 시간 간격: 8ms
           
Window 2:            ├─ 0.016 ~ 0.032 초
                     │
                     └─ 시간 간격: 8ms
                     
...

Window 153:                    ├─ 1.2315 ~ 1.2475 초
                               │
═══════════════════════════════════════════════════════════════

실제 예시:

Window 0: [제조 기계의 진동 신호 0-16ms 구간]
           [정상 운영 중의 진동]
           array[0] = [[진동값], [진동값], ..., [진동값]] # 64개
           
Window 50: [제조 기계의 진동 신호 400-416ms 구간]
           [여전히 정상 운영]
           array[50] = [[진동값], [진동값], ..., [진동값]] # 64개
           
Window 120: [제조 기계의 진동 신호 960-976ms 구간]
            [이상 신호 발생?]
            array[120] = [[높은진동], [높은진동], ..., [높은진동]] # 64개
            [진동값이 대체로 큼 - 이상 가능성]
"""

"""

                  # 이는 Window 50의 정확히 중간 시점의 진동값
"""
