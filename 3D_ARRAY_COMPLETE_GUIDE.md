# 전처리 후 3D Array: 완전 가이드

## 📊 빠른 요약

**질문**: 전처리 후 3D Array는 어떤 정보를 가지고 있어?

**답변**: 
```
3D Array Shape: (154, 64, 1)

└─ 154: 시간 윈도우 개수 (뉘움-구간별 데이터)
   └─ 64: 각 윈도우의 샘플 개수 (시간축 길이) 
      └─ 1: 특성 개수 (진동 신호만)
         └─ 정규화된 값 [-3, 3]: 평균으로부터의 표준편차
```

---

## 🔍 상세 설명

### 1. 배열의 각 차원

#### 차원 1: 윈도우 개수 (154)

**의미**:
- 전체 원본 데이터를 겹치는 32-샘플 간격으로 분할한 결과
- 4990개 원본 샘플 → 154개 윈도우

**계산식**:
```
n_windows = floor((total_samples - window_size) / step_size) + 1
         = floor((4990 - 64) / 32) + 1
         = floor(154.5625)
         = 154 윈도우
```

**각 윈도우의 인덱스**:
- 0번 윈도우: 원본 샘플 0~63 (시간 0~15.75ms)
- 1번 윈도우: 원본 샘플 32~95 (시간 8~23.75ms)
- 2번 윈도우: 원본 샘플 64~127 (시간 16~31.75ms)
- ...
- 153번 윈도우: 원본 샘플 4926~4989 (시간 1232~1248ms)

**시각화**:
```
원본 데이터 타임라인:
[s0][s1]...[s63]
  └─ Window 0

          [s32][s33]...[s95]
           └─ Window 1

                    [s64][s65]...[s127]
                     └─ Window 2
                     
                                ...
                                
                                    [s4926][s4927]...[s4989]
                                     └─ Window 153

각 윈도우는 64개 연속 샘플
인접 윈도우는 32개 샘플씩 겹침 (50% overlap)
```

---

#### 차원 2: 윈도우 내 샘플 개수 (64)

**의미**:
- 각 윈도우 내의 시간 기점(time steps)
- 시간을 64개 균등 구간으로 샘플링

**시간 범위**:
- Config의 `WINDOW_SIZE = 64`
- 샘플 레이트: 4000 Hz
- 시간당 64개 샘플: 64 / 4000 = 0.016초 = **16 ms**

**각 샘플의 시간 위치**:
```
array[w][0] → 윈도우 w의 시작 시점 (t=0)
array[w][1] → 1 샘플 후 (t=0.25ms)
array[w][2] → 2 샘플 후 (t=0.5ms)
...
array[w][63] → 63 샘플 후 (t=15.75ms) ← 윈도우 끝

각 시점: Δt = 1/4000 = 0.25 ms 간격
```

**예시**:
```
Window 10의 시간 범위:
- 시작: 10 × 32 샘플 × 0.25ms = 80ms
- 끝: 80ms + 16ms = 96ms
- 범위: [80ms ~ 96ms]

Window 10의 샘플들:
array[10][0] → 80.00ms의 진동값
array[10][1] → 80.25ms의 진동값
array[10][2] → 80.50ms의 진동값
...
array[10][63] → 95.75ms의 진동값
```

---

#### 차원 3: 특성 개수 (1)

**의미**:
- 각 시점에서 측정된 입력 변수 개수
- 현재는 1개만: **vibration** (진동 신호)

**구조**:
```
array[w][s][0]
              ↑ 0번 인덱스 = vibration 특성

만약 다른 센서가 추가되면:
array[w][s][0] → vibration (진동)
array[w][s][1] → temperature (온도) 
array[w][s][2] → pressure (압력)
...
```

**현재 설정** (config.py):
```python
USE_STATISTICAL_FEATURES = False  # 비활성화 상태
# 활성화하면 RMS, Peak, Crest, Kurtosis, Skewness 추가
# 그러면 n_features = 1 + 5 = 6

USE_FFT_FEATURES = False  # 비활성화 상태  
# 활성화하면 FFT 32개 빈 추가
```

### 2. 데이터 정규화

**전처리 파이프라인**:
```
1. CSV 로드
   └─ vibration: [-0.0065, -0.0184, -0.0095, ...]
   
2. 결측치 처리
   └─ 선형 보간법 (현재 데이터는 결측치 없음)
   
3. 이상치 처리
   └─ IQR 방법으로 범위 외 값 제거/클립
   
4. 정규화 (StandardScaler)
   └─ x_norm = (x - mean) / std
   └─ 결과: [-0.0234, -0.0667, -0.0344, ...]
   
5. 윈도우 생성
   └─ 정규화 데이터를 64-샘플 윈도우로 분할
   └─ 최종 3D Array (154, 64, 1)
```

**정규화 특성**:
```
원본 데이터:
  범위: 자유로움 (예: -0.5 ~ 15.3)
  평균: 특정 값 (예: 2.1)
  표준편차: 일정 (예: 3.2)

정규화 후:
  평균: 0 (정확히)
  표준편차: 1 (정확히)
  범위: [-3 ~ 3] (약 95% 데이터)
```

**값의 의미**:
```
값      의미
─────────────────────────────
0       평균 진동 (정상)
+1.0    평균 + 1σ
+2.0    평균 + 2σ (이상 신호 가능)
+3.0    평균 + 3σ (강한 이상 신호)
-1.0    평균 - 1σ
-2.0    평균 - 2σ (이상 신호 가능)
-3.0    평균 - 3σ (강한 이상 신호)
```

### 5. 각 윈도우가 나타내는 의미

**시계열 관점**:
```
각 윈도우 = 모터의 연속된 16ms 진동 신호

Window 0:  [정상 모터의 0~16ms 진동]
           모터가 정상 작동 중

Window 50: [정상 모터의 400~416ms 진동]
           여전히 정상 작동

Window 120: [이상 모터의 960~976ms 진동]
            높은 진동값 → 이상 신호 감지
            예: array[120] 값들이 대체로 > 1.5
```

**신호 해석**:
```
array[w] 의 특성:
- 평균이 높음 (> 1.5)    → 이상 진동
- 평균이 -1.5 이하      → 역방향 이상
- 표준편차가 높음        → 불안정한 신호
- 값이 [-3, 3] 범위 내  → 정상 범위
```

## 🎯 핵심 요약

### 3D Array는 이렇게 구성됨:

```
전체 시계열 데이터 (1248ms, 4990샘플)
    ↓
    겹치는 윈도우로 분할 (64-샘플 윈도우, 32 스텝)
    ↓
    154개 윈도우 생성
    ├─ 각 윈도우: 16ms의 연속 신호
    ├─ 64개 시점: 0.25ms 간격
    └─ 1개 특성: 정규화된 진동값
    ↓
    최종: 3D Array (154, 64, 1)
```

### 각 값(element)은:
- **배치**: 154개 시간 구간 중 하나
- **시계열**: 64개 연속 시점의 신호
- **값**: 정규화된 진동 진폭 (-3~3)

### 활용:
- **이상 탐지**: 높은/낮은 진동값 감지
- **패턴 인식**: LSTM에서 시계열 패턴 학습
- **특성 추출**: classical ML에서 64개 값을 피처로 사용
